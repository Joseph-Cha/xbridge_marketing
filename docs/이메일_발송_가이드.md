# 이메일 발송 시스템 사용 가이드

수집된 업체 이메일 주소로 xBridge 랜딩 페이지 안내 이메일을 발송하는 시스템입니다.
Resend API를 사용하여 SMTP 설정 없이 이메일을 발송합니다.

---

## 파일 구조

```
scripts/
├── email_config.py        # Resend API 키, 발송 속도, 파일 경로 등 설정
├── email_template.py      # HTML/텍스트 이메일 템플릿
├── send_emails.py         # 이메일 발송 메인 스크립트
├── manage_recipients.py   # 수신자 관리 (현황 조회, CSV 병합, 수신 거부)
└── output/
    ├── 업체_이메일_결과.csv       # 수신자 목록 (scrape_emails.py 결과)
    ├── 이메일_발송_결과.csv       # 발송 로그 (자동 생성)
    └── 수신거부_목록.csv          # 수신 거부 목록 (자동 생성)
```

## 1단계: Resend 가입 및 설정

### 1-1. 회원가입

1. [Resend 가입 페이지](https://resend.com/signup) 접속
2. GitHub 또는 이메일로 회원가입

### 1-2. 도메인 등록 및 인증

자사 도메인(예: `xbridge.kr`)으로 이메일을 발송하려면 도메인 인증이 필요합니다.

1. [Resend Domains](https://resend.com/domains) 페이지 접속
2. **Add Domain** 클릭 후 도메인 입력 (예: `xbridge.kr`)
3. 표시되는 DNS 레코드를 도메인 관리 페이지(가비아, 클라우드플레어 등)에 추가:
   - **SPF** 레코드 (TXT)
   - **DKIM** 레코드 (TXT) - 3개
   - **DMARC** 레코드 (TXT) - 선택사항이지만 권장
4. DNS 반영 후 Resend에서 **Verify** 클릭 (반영까지 수분~수시간 소요)

> **테스트만 할 경우:** 도메인 인증 없이도 `onboarding@resend.dev`를 발신자로 사용하여 테스트 발송이 가능합니다. 단, 본인 계정 이메일로만 수신됩니다.

### 1-3. API 키 발급

1. [Resend API Keys](https://resend.com/api-keys) 페이지 접속
2. **Create API Key** 클릭
3. 이름: `xBridge Outreach` 등 입력
4. Permission: **Sending access** 선택
5. 생성된 `re_` 로 시작하는 키를 복사

### 1-4. 설정 파일에 입력

`scripts/email_config.py`를 열고 다음 항목을 입력합니다:

```python
RESEND_API_KEY = "re_xxxxxxxx..."        # 발급받은 API 키

SENDER_NAME = "xBridge"
SENDER_EMAIL = "noreply@xbridge.kr"      # 인증한 도메인의 이메일 주소
# 테스트 시: SENDER_EMAIL = "onboarding@resend.dev"
```

### Resend 무료 플랜 한도

| 항목 | 한도 |
|---|---|
| 일일 발송 | 100건 |
| 월간 발송 | 3,000건 |
| 커스텀 도메인 | 1개 |

> 현재 발송 대상이 약 192건이므로, 무료 플랜으로 2일에 걸쳐 전체 발송이 가능합니다.

## 3단계: 발송 대상 확인 (Dry Run)

실제 발송 전에 대상 목록을 반드시 확인합니다.

```bash
# 전체 발송 대상 확인
python scripts/send_emails.py --dry-run

# 상위 10건만 미리보기
python scripts/send_emails.py --dry-run --limit 10
```

출력 예시:

```
============================================================
  xBridge 이메일 발송 시스템 (Resend API)
============================================================
  전체 수신 대상: 192건
  이미 발송됨:    0건 (스킵)
  이번 발송 대상: 50건
  모드: DRY-RUN (실제 발송 없음)
============================================================

  #    업체명                                   이메일
  ---- ---------------------------------------- ----------------------------------------
  1    AGRICULTURAL HAEOREUM FOOD INC.          tnrgus1771@hanmail.net
  2    AGRICULTURAL JANGHEUNG FOOD CO.          djchun0425@hanmail.net
  ...
```

## 4단계: 테스트 발송

본인 이메일로 먼저 테스트하여 내용과 레이아웃을 확인합니다.

```bash
python scripts/send_emails.py --test-to your@email.com
```

출력 예시:

```
  테스트 발송 (Resend API)
  발신자: xBridge <noreply@xbridge.kr>
  수신자: your@email.com
  제목: [xBridge] AI 기반 K-Food 수출 플랫폼, 사전 신청 안내

  [성공] 테스트 이메일이 your@email.com로 발송되었습니다.
  Resend 대시보드에서 확인: https://resend.com/emails
```

이메일이 정상 수신되면 HTML 렌더링, CTA 버튼 링크 동작 여부를 확인합니다.
발송 상태는 [Resend 대시보드](https://resend.com/emails)에서도 실시간 확인 가능합니다.

## 5단계: 실제 발송

```bash
# 소량 발송 (처음에는 10건으로 시작 권장)
python scripts/send_emails.py --limit 10

# 기본 발송 (BATCH_SIZE=50건씩)
python scripts/send_emails.py

# 전체 발송 (email_config.py에서 BATCH_SIZE=0으로 변경 후)
python scripts/send_emails.py
```

> **일일 100건 제한 참고:** 무료 플랜에서는 하루 100건까지만 발송됩니다.
> `--limit 100`으로 설정하고 다음 날 다시 실행하면 이미 발송된 건은 자동 스킵됩니다.

발송 중 화면에 진행 상황이 표시됩니다:

```
  발신자: xBridge <noreply@xbridge.kr>

  [O] 1/10 AGRICULTURAL HAEOREUM FOOD INC. -> tnrgus1771@hanmail.net
  [O] 2/10 AGRICULTURAL JANGHEUNG FOOD CO. -> djchun0425@hanmail.net
  [X] 3/10 SOME COMPANY -> invalid@email.com (Resend API 오류: ...)
```

- `[O]` = 발송 성공
- `[X]` = 발송 실패 (사유 표시)

## 신규 업체 리스트 추가

추후 새로운 업체 이메일을 확보했을 때 기존 목록에 병합합니다.

### 방법 1: CSV 병합

신규 CSV 파일의 형식 (최소 필수 열):

```csv
업체명,웹사이트,이메일,상태
새로운 회사,http://example.kr,info@example.kr,성공
```

병합 실행:

```bash
python scripts/manage_recipients.py --merge scripts/output/신규_업체.csv
```

- 기존 목록에 없는 업체만 추가됩니다 (업체명 기준 중복 제거)
- 병합 후 바로 `send_emails.py`로 발송 가능 (이미 발송된 건은 자동 스킵)

### 방법 2: 별도 CSV 파일로 직접 발송

기존 목록에 병합하지 않고 신규 파일로 바로 발송할 수도 있습니다.

```bash
python scripts/send_emails.py --csv scripts/output/신규_업체.csv --dry-run
python scripts/send_emails.py --csv scripts/output/신규_업체.csv
```

## 실패 건 재발송

발송 실패한 건만 선택적으로 재시도합니다.

```bash
python scripts/send_emails.py --retry-failed
```

## 수신 거부 관리

수신 거부 요청이 들어오면 해당 이메일을 등록합니다.

```bash
# 수신 거부 등록
python scripts/manage_recipients.py --unsubscribe someone@company.com

# 수신 거부 목록 확인
python scripts/manage_recipients.py --list-unsubscribed
```

등록된 이메일은 이후 발송 시 자동으로 제외됩니다.

## 설정 커스터마이즈

`scripts/email_config.py`에서 조절 가능한 주요 설정:

| 설정 | 기본값 | 설명 |
|---|---|---|
| `RESEND_API_KEY` | (비어있음) | Resend API 키 (`re_` 로 시작) |
| `SENDER_EMAIL` | (비어있음) | 발신자 이메일 (인증된 도메인 필요) |
| `SEND_DELAY` | 2 | 이메일 간 대기 시간(초). Rate limit 방지용 |
| `BATCH_SIZE` | 50 | 한 번 실행 시 최대 발송 건수. 0이면 무제한 |
| `RETRY_COUNT` | 2 | 발송 실패 시 재시도 횟수 |
| `RETRY_DELAY` | 5 | 재시도 대기 시간(초) |
| `EMAIL_SUBJECT` | `[xBridge] AI 기반...` | 이메일 제목 |
| `LANDING_PAGE_URL` | `https://xbridge.kr` | CTA 버튼 링크 URL |

## 이메일 템플릿 수정

`scripts/email_template.py`에서 이메일 본문을 수정할 수 있습니다.

- `build_email_html()` — HTML 버전 (대부분의 이메일 클라이언트)
- `build_email_text()` — 텍스트 버전 (HTML 미지원 클라이언트용 대체)

템플릿에서 `{company_name}`은 수신 업체명으로, `{landing_page_url}`은 랜딩 페이지 URL로 자동 치환됩니다.

## 주의사항

- **소량부터 시작**: 처음에는 `--limit 10` 정도로 소량 발송 후 반응을 확인하세요
- **일일 한도 준수**: Resend 무료 플랜은 일 100건 / 월 3,000건 제한입니다
- **도메인 인증 필수**: 실제 업체에 발송하려면 자사 도메인 인증을 완료해야 합니다
- **수신 거부 즉시 처리**: 수신 거부 요청은 반드시 즉시 등록해야 합니다
- **중복 발송 없음**: 한 번 성공적으로 발송된 이메일은 재실행해도 다시 보내지 않습니다
- **대시보드 활용**: [Resend 대시보드](https://resend.com/emails)에서 발송 상태, 열람률, 바운스 등을 모니터링하세요
