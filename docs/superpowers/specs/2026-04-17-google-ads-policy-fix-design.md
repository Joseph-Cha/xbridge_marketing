# Google Ads 정책 거절 대응 — 랜딩 페이지 리포지셔닝 디자인

> 작성일: 2026-04-17
> 상태: 디자인 승인 대기
> 관련 PRD: `docs/PRD.md`

---

## 1. 배경

XBridge 사전 신청 랜딩 페이지가 Google Ads 심사에서 거절됨.

**거절 사유 (Google 통보)**
> *Destinations that display a message of the destination not providing any services*
> Examples: Parked domains, Pages that show "Under construction" or "Coming soon", Pages with no content

해당 정책: *Destination Requirements > Destination not working*

**현 페이지에서 정책에 직접 걸리는 시그널**
- Hero 배지 `2026년 상반기 출시 예정`
- Hero 본문 `K-Food 수출 플랫폼이 곧 출시됩니다`
- 모든 CTA가 `사전 신청`
- Benefits 4개 카드가 모두 *출시 후* 받게 될 혜택 (우선 초대 / 첫 3개월 50% 할인 / 베타 테스터)
- Features 3개 카드가 "서비스 *예정* 기능"
- 약관 1조 `사전 신청 서비스`, 3조 `정식 출시 시 우선 초대`

CSS로 배지만 가리는 식의 회피로는 통과 불가. 페이지의 본질적 가치 제안 자체가 *"지금은 서비스가 없고 나중에 출시됨"* 으로 읽히기 때문에, 본질을 바꿔야 함.

## 2. 부수 위험 (같은 작업에서 같이 해소)

같은 페이지 내에 다음 추가 거절 위험 요소가 있어 1차 수정과 함께 정리한다.

| 항목 | 정책 위험 | 현재 상태 |
|---|---|---|
| 가격 약속의 불투명성 | *Misrepresentation > Unclear pricing* | "유료 플랜 첫 3개월 50% 할인" — 정식 가격 미공개 |
| 사업자 신원 정보 부재 | *Misrepresentation > Unidentified business* + 한국 전자상거래법 | Footer에 법인명·사업자번호·주소·연락처 모두 없음 |
| 위탁업체 사실 불일치 | *Misrepresentation* + 개인정보보호법 | Privacy Page에 `Google Sheets`로 표기, 실제 코드는 Supabase |
| 거짓 희소성 | *Misrepresentation* | "선착순 500개 기업 한정" 압박 문구 |
| 검증 안 된 파트너 표기 | *Misrepresentation* | TrustSection 협력 기관 4곳 — 실제 협약 검증 필요 |

## 3. 핵심 디자인 — A안 리포지셔닝

페이지의 본질을 바꾼다.

| | Before | After |
|---|---|---|
| **본질** | 출시 예정 플랫폼의 사전 신청 | 지금 제공 중인 K-Food 수출 무료 컨설팅 + 규제 진단 리포트 |
| **메인 오퍼** | 미래 혜택 (출시 시 50% 할인 등) | 현재 서비스: 1:1 통화 컨설팅 + 4대 규제 진단 리포트(PDF) |
| **CTA 동사** | "사전 신청" | "무료 진단 받기" |

### 3.1 약속 명세 (페이지에 그대로 노출되는 SLA)

| 항목 | 약속 |
|---|---|
| 신청 → 전문가 연락 | **영업일 2일 내** 담당 전문가가 직접 1:1 통화 |
| 통화 → 진단 리포트 PDF | 통화 후 **영업일 5일 내** 메일 발송 |
| 비용 | **100% 무료**, 결제 정보 일절 요구 안 함 |

### 3.2 진단 리포트 구성 (4대 규제 영역)

신청서의 `industry`(식품 카테고리)와 `target_countries`(관심 수출국) 입력값을 기반으로 다음 4개 영역을 각 진출국 기준으로 분석한 PDF를 발송한다.

1. **금지/제한 성분 목록** — 해당 국가에서 금지/제한된 식품 성분 및 첨가물
2. **식품 라벨링 요건** — 표시사항, 언어, 영양정보, 원산지 표기 규정
3. **식품 안전 기준** — 미생물·중금속·잔류농약 등 안전 기준
4. **수입 절차 및 필수 서류** — 수입 신고, 인증, 검역, 통관 서류

→ 전문가가 신청 정보 검토 후 작성. 자동 생성 아님.

## 4. 컴포넌트별 변경 명세

### 4.1 `HeroSection.tsx`
- 배지: `2026년 상반기 출시 예정` → **`K-Food 수출 전문가 1:1 무료 진단`**
- 헤드라인: 유지 가능 (`K-Food, 세계 시장으로 연결하는 가장 확실한 길`)
- 서브카피: `AI가 최적 시장을 찾아주고, 검증된 바이어까지 연결해 주는 K-Food 수출 플랫폼이 곧 출시됩니다.` → **`10년 경력 K-Food 수출 전문가가 1:1 통화로 진단하고, 진출국 4대 규제 분석 리포트(PDF)를 무료로 보내드립니다.`**
- CTA 라벨: `사전 신청하고 특별 혜택 받기` → **`무료 수출 규제 진단 받기`**
- CTA 보조 문구: `지금 신청하시면 출시 시 우선 초대 + 특별 할인 혜택` → **`100% 무료 · 영업일 2일 내 담당 전문가 직접 연락`**
- 카운터 문구: `현재 X개 기업이 사전 신청 완료` → **`현재 X개 기업이 무료 진단 신청`** (수치 데이터 그대로 사용)
- "출시 예정" 시각 시그널(Rocket 아이콘) → MessageCircle / FileText 등 컨설팅 친화 아이콘으로 교체

### 4.2 `FeaturesSection.tsx`
"서비스 예정 기능" 3개 카드 전부 삭제. **컨설팅 진행 절차 4스텝**으로 교체.

| 스텝 | 제목 | 설명 |
|---|---|---|
| 01 | 진단 신청서 작성 | 약 3분 소요 — 회사 정보 + 진출 희망국 |
| 02 | 1:1 전문가 통화 | 영업일 2일 내 담당 전문가 직접 연락 |
| 03 | 진단 리포트 PDF 발송 | 통화 후 영업일 5일 내 메일로 |
| 04 | 후속 액션 안내 | 필요 시 추가 상담·실행 지원 |

→ 섹션 제목도 `서비스 예정 기능` → **`무료 진단 진행 절차`** 로 변경.

### 4.3 `BenefitsSection.tsx`
4개 카드를 **3개 카드**로 교체 (실제 무료 제공 항목만).

| | Before (삭제) | After (실제 제공) |
|---|---|---|
| 1 | 우선 초대권 | **1:1 전문가 통화** (10년+ K-Food 수출 경력 전문가) |
| 2 | 첫 3개월 50% 할인 | **4대 규제 진단 리포트 (PDF)** — 금지 성분 / 라벨링 / 안전 기준 / 수입 절차 |
| 3 | 무료 컨설팅 | **진단 결과 기반 액션 가이드** — 다음 단계 우선순위 권고 |
| 4 | 베타 테스터 | (삭제) |

- 섹션 헤더: `지금 사전 신청하시면` → **`무료로 받게 되는 것`**
- 서브카피: `정식 출시 전 신청자에게만 드리는 특별 혜택` → **`신청 한 번으로 받는 K-Food 수출 무료 진단 패키지`**
- **하단 "선착순 500개 기업 한정" 배지 완전 삭제** (거짓 희소성 위험)

### 4.4 `TrustSection.tsx`
- "준비하고 있어요" / "함께 만들어요" → **현재형 "제공합니다" / "직접 컨설팅합니다"** 로 변경
- "검증된 기술로 수출 성공률을 높여드려요" → 현재 진행 중 컨설팅 톤으로 조정 (예: "10년 경력 전문가의 시장·규제 분석으로 수출 진입 리스크를 낮춥니다")
- **"파트너 협력" 영역**: 실제 협약/MOU 또는 거래 관계가 검증 가능한 곳만 남기기. 검증 불가한 곳은 *misrepresentation* 위험이므로 삭제. → **유저 액션 필요**: 4곳(`한국무역협회`, `Pinesol VINA`, `DYPrime`, `록시에이에프씨`) 각각의 검증 가능 여부 확인 후 구현 시 정리.

- 폼 필드 구성 + Zod 검증 — 그대로
- UTM 추적 / GA4 이벤트 — 그대로 (이벤트명 `form_submit`, `cta_click` 등 라벨만 의미가 바뀜)
- Supabase RLS, 환경변수, 배포 설정 — 그대로
- Hero 카운터 (`/api/leads/count`) 데이터 소스 — 그대로 (라벨만 변경)
